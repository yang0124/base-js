1、*****函数
   什么是函数：程序中描述一项任务的步骤清单的代码段
	       代码段起一个名字
   何时使用：只要一段代码 或者 一项任务可能被反复执行时
	     都要先定义为一个函数，再反复调用函数执行
   
   如何使用：
	1、先创建函数，以*声明*方式创建函数
	  *function 函数名(形参列表){
		函数体;
		return 返回值/函数的结果;
	   }
	   形参列表：专门接受函数执行时，必要数据的变量
	     何时使用：如果一个函数，必须接受指定的数据才能正常运算时
	     如何定义：不用var创建，每个形参变量之间用逗号分隔
	   返回值：函数的执行结果
	     何时使用：如果将来调用函数的人需要获得函数的执行结果时，函数就要将执行结		               果返回
	     如何返回：return 返回值;
		 return：本意其实是退出函数
			 如果return后跟着一个数据，就会顺便将数据返回到内存！
			 只负责返回，不负责保存

	2、调用函数执行，获得函数的执行结果
	   var 返回值变量 = 函数名(实参列表);
	   实参列表：一个数据 或者 多个数据，逗号分隔
	     实参的个数不一定和形参的个数一致
		如果实参的个数>形参的个数
		  就舍弃多余的实参
		如果实参的个数<形参的个数
		  没有获得实参的形参，默认值为undefined
	     建议：严格按照形参的个数传入实参
	   返回值变量：专门用于接收函数调用结果的变量
		       如果不需要接住函数调用结果，可以不写

****作用域（scope）：一个变量的可用范围
    1、全局作用域：在全局作用域中存储的变量，在程序的任何位置都可以访问
	 全局作用域中创建的函数，都称之为全局变量
	 函数名其实也是一个变量
	 函数定义其实是一个引用类型的对象
	 函数还可定义为：使用*函数直接量方式创建函数*
		var 函数名=function(形参列表){函数体;return 返回值;}
      
    2、函数作用域：在函数作用域中存储的变量，只能在本次函数调用时，才能内部访问
	 *变量的使用规则：优先使用局部变量
			 如果局部没有，才到全局找
			 如果全局没有，就报错
       ***局部变量：包括 形参 和 在函数内var创建的变量
	 在任意位置给未声明的变量赋值，都会在全局创建该变量
		-- 禁止！
	 全局污染：在局部尝试给未声明的变量直接赋值，导致篡改全局的变量值
	 强烈建议：所有变量必须var声明后，才能使用

	 *直接使用一个未声明的变量，会报错
	 *直接给一个未声明的变量赋值，会全局污染

****声明提前（hoist）：
    在程序执行前
    将var声明的变量和function声明的函数
    集中到*当前作用域*的开始位置创建
    赋值留在原地

    强调：只有声明方式创建的函数才能被完整的提前
	  直接量方式创建的函数，只有声明变量部分会提前

    如果以后鄙视面试时：遇到了先使用，后创建
	一定是再考声明提前
	先变成提前后的样子，然后再去判断输出

    我们写代码永远不会碰到：只要遵守先创建后使用

上午的练习：
    1、尝试全局变量和局部变量的关系
    2、尝试在局部给一个未声明的变量直接赋值，看看全局会怎么样
    3、随意的写一些声明提前，然后让你的同桌来判断

****按值传递：两个变量之间赋值
    如果传递的是原始类型的值
	修改新变量，原变量是不受影响的，其实是复制了一个副本给对方
	修改副本跟自己没关系

    如果传递的是引用类型的对象
	通过新变量修改对象，等效于直接修改对象本身，因为两个用的是同一个地址值

   1、编码解码
      问题1：url中不允许出现多字节字符，如果出现会乱码
	     utf-8,1个汉字，3字节
      解决：发送前，将多字节内容编码为单字节内容
            收到后，再将单字节内容解码为多字节原文

      如何编码解码：
	编码：var code=encodeURI("原文");
	解码：var 原文=decodeURI(code);

      问题2：url中不允许出现保留字符：比如:  / :
        解决：用 encodeURIComponent 代替 encodeURI
	      用 decodeURIComponent 代替 decodeURI

   4、switch...case...
      如何使用
      switch(变量/表达式){
	case 值1:
	操作1;
	case 值2:
	操作2;
	case 值3:
	操作3
	default:
	默认操作;
      }
      特点：1、默认如果满足了某个条件，会将后续所有操作全部做完，可以使用break
	    2、何时可以不写break：
		如果多个条件做的是相同操作，中间的break和中间的操作都可以省略
   		最后default不需要break

	if vs switch
	1、switch想要使用必须知道，结果是什么，才能去判断，不能取范围
	2、switch的效率更高，因为不需要判断范围
	3、if一定是用的最多的，如果简单的if其实可以用三目代替
